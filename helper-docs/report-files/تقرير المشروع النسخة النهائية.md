# نظام استرجاع معلومات متقدم

## 1. نظرة عامة على المشروع

هذا المشروع هو نظام استرجاع معلومات (Information Retrieval System) كامل ومتكامل، تم بناؤه باستخدام لغة Python وتوظيف بنية الخدمات المنفصلة (SOA). يهدف النظام إلى توفير منصة قوية للبحث في مجموعات بيانات نصية كبيرة باستخدام مجموعة متنوعة من النماذج الكلاسيكية والمتقدمة، مع توفير ميزات إضافية لتحسين دقة البحث وتجربة المستخدم.

النظام قادر على معالجة وبناء نماذج لمجموعات بيانات متعددة، ويتيح للمستخدم الاختيار بينها والبحث فيها عبر واجهة مستخدم رسومية بسيطة وفعالة.

---

## 2. الميزات والتقنيات المستخدمة

### المتطلبات الأساسية

- **بنية الخدمات المنفصلة (SOA):** تم تصميم النظام ليكون قابلاً للتوسع والصيانة، حيث تعمل كل مهمة رئيسية (تحميل البيانات، بناء النماذج، البحث) كخدمة API مستقلة.
- **تخزين فعال:** استخدام قاعدة بيانات MySQL لتخزين المستندات، مما يضمن أداءً وسرعة أفضل من التعامل مع الملفات.
- **معالجة نصوص متقدمة:** تطبيق خط أنابيب لمعالجة النصوص يتضمن:
    - Tokenization
    - Lemmatization (للحفاظ على المعنى)
    - إزالة الكلمات المستبعدة وعلامات الترقيم.
- **نماذج تمثيل متعددة:**
    - **TF-IDF:** مع استخدام الفهرس المعكوس لتحسين سرعة البحث.
    - **BM25:** النموذج الإحصائي القوي مع إمكانية ضبط معاملاته (`k1`, `b`).
    - **BERT Embeddings:** استخدام نماذج Transformers (`all-MiniLM-L6-v2`) لفهم المعنى الدلالي.
    - **Vector Store (FAISS):** استخدام فهرس FAISS للبحث فائق السرعة في متجهات BERT.
    - **نموذج هجين (Hybrid):** دمج نتائج BM25 و BERT باستخدام تقنية المجموع الموزون (Weighted Sum) لتحقيق أفضل أداء.

### المتطلبات الإضافية

- **إعادة ترتيب النتائج بـ NER:** تطبيق آلية بحث من مرحلتين تستخدم "التعرف على الكيانات المسماة" (NER) لإعطاء وزن إضافي للنتائج التي تحتوي على أسماء وكيانات تطابق الاستعلام.
- **اقتراح الاستعلام (Query Suggestion):** توفير ميزة الإكمال التلقائي في مربع البحث لمساعدة المستخدم على صياغة استعلامه، وذلك بالاعتماد على الفهرس المعكوس.
- **تجميع المستندات (Document Clustering):** استخدام خوارزمية `K-Means` على تمثيلات TF-IDF لتصنيف المستندات إلى "مواضيع" أو "عناقيد" متشابهة.

## 3. البنية المعمارية للنظام (SOA)

يتكون النظام من 4 خدمات رئيسية مستقلة:

1.  **`data_loader_api` (منفذ 8001):**
    * **المسؤولية:** استقبال طلبات تحميل ومعالجة مجموعات البيانات.
    * **آلية العمل:** تقرأ البيانات من `ir_datasets`، تطبق عليها المعالجة الأولية عبر `TextPreprocessor`، ثم تخزن النصوص الأصلية والمعالجة في قاعدة بيانات MySQL على دفعات.

2.  **`representation_api` (منفذ 8002):**
    * **المسؤولية:** بناء جميع النماذج والفهارس وحفظها.
    * **آلية العمل:** تستخدم استراتيجية محسنة للذاكرة؛ حيث تبني نماذج TF-IDF و BM25 باستخدام تقنية "الجريان" (Streaming) لتجنب استهلاك الذاكرة، ثم تبني فهارس BERT/FAISS و Inverted Index على دفعات. كما تحتوي على Endpoint إضافي لبناء عناقيد المستندات.

3.  **`search_api` (منفذ 8003):**
    * **المسؤولية:** هي "العقل" المفكر للنظام. تستقبل طلبات البحث والاقتراحات.
    * **آلية العمل:** تقوم بتحميل النماذج المحفوظة، تنفذ البحث باستخدام النموذج المختار، تطبق آلية إعادة الترتيب بـ NER (إذا تم تفعيلها)، وتعيد النتائج. كما تحتوي على Endpoint للبحث في الفهرس المعكوس وتقديم اقتراحات.

4.  **`ui` (منفذ 5001):**
    * **المسؤولية:** توفير واجهة المستخدم الرسومية (Web UI).
    * **آلية العمل:** تطبيق Flask بسيط يعرض صفحة `index.html`. يقوم JavaScript في الصفحة بالتخاطب مع `search_api` لعرض نتائج البحث والاقتراحات للمستخدم.
