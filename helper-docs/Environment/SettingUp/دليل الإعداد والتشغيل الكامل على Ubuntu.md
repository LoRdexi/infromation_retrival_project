### الخطوة 0: التعديل الأهم (قبل كل شيء)

- افتح ملف **`config.py`**.
    
- قم بتعديل قاموس `DB_CONFIG` ليتوافق مع إعدادات MySQL على جهازك (اسم المستخدم، كلمة المرور، اسم قاعدة البيانات). هذا هو أهم تعديل يجب القيام به.
    

### الخطوة 1: إعداد البيئة وقاعدة البيانات

1. **تثبيت MySQL Server (إذا لم يكن مثبتاً):**
    
    ```
    sudo apt update
    sudo apt install mysql-server
    # بعد التثبيت، قم بتشغيل سكربت الإعداد الآمن
    sudo mysql_secure_installation
    # اتبع التعليمات لتعيين كلمة مرور لمستخدم root.
    ```
    
2. **إنشاء قاعدة البيانات والمستخدم:**
    
    ```
    # الدخول إلى MySQL
    sudo mysql
    # قم بتنفيذ الأوامر التالية (استبدل 'your_password' بكلمة المرور التي اخترتها في config.py)
    CREATE DATABASE IF NOT EXISTS ir_project_db_v2;
    CREATE USER IF NOT EXISTS 'ir_user'@'localhost' IDENTIFIED BY 'your_password';
    GRANT ALL PRIVILEGES ON ir_project_db_v2.* TO 'ir_user'@'localhost';
    FLUSH PRIVILEGES;
    EXIT;
    ```
    
3. **إعداد بيئة بايثون الافتراضية:**
    
    ```
    # إنشاء البيئة الافتراضية
    python3 -m venv venv
    source venv/bin/activate
    # تثبيت المكتبات المطلوبة
    pip install -r requirements.txt
    ```
    
4. **إنشاء جداول قاعدة البيانات:**
    
    ```
    # سيقوم هذا السكربت بإنشاء الجداول اللازمة في قاعدة البيانات التي حددتها
    python utils/database_setup.py
    ```
    

### الخطوة 2: تشغيل الخدمات الخلفية (Backend APIs)

1. **تثبيت أداة مساعدة (إذا لم تكن مثبتة):**
    
    ```
    # هذا السكربت يستخدم أمر fuser لإيقاف أي خدمات قديمة. قد تحتاج لتثبيته.
    sudo apt update && sudo apt install psmisc
    ```
    
2. **جعل سكربت التشغيل قابلاً للتنفيذ:**
    
    ```
    chmod +x run_services.sh
    ```
    
3. **تشغيل جميع الخدمات:**
    
    ```
    ./run_services.sh
    ```
    
    - سيقوم هذا الأمر بتشغيل جميع الخدمات الثلاث **في الخلفية**.
        
    - سيتم إنشاء ملفات سجلات (`.log`) لكل خدمة، وهي مهمة جداً لمراقبة التقدم.
        
    - **لمراقبة السجلات:** افتح طرفية جديدة واستخدم الأمر: `tail -f *.log`
        
    - **لإيقاف الخدمات:** استخدم الأمر `kill` مع أرقام العمليات (PIDs) التي يعرضها السكربت.
        

### الخطوة 3: معالجة البيانات وبناء النماذج (عبر Postman)

الآن بعد أن أصبحت الخدمات تعمل، يجب أن نطلب منها القيام بالمهام الثقيلة.

1. **افتح برنامج Postman** وقم باستيراد ملف `IR_Project_SOA.postman_collection.json`.
    
2. **أرسل الطلبات بالترتيب التالي:**
    
    - **أولاً: `[Step 1] Load and Process Dataset`**
        
        - أرسل هذا الطلب (POST) لبدء تحميل ومعالجة مجموعة البيانات التي تختارها في الـ Body.
            
        - **انتظر حتى تكتمل العملية.** يمكنك متابعة التقدم عبر مراقبة ملف `data_loader.log`.
            
    - **ثانياً: `[Step 2] Build Representations`**
        
        - **فقط بعد انتهاء الخطوة السابقة**، أرسل هذا الطلب (POST) لبناء جميع النماذج والفهارس.
            
        - هذه العملية قد تستغرق وقتاً طويلاً. راقب ملف `representation.log` لمعرفة متى تنتهي.
            

### الخطوة 4: تشغيل واجهة المستخدم

بعد اكتمال الخطوات السابقة بنجاح، أنت جاهز لتشغيل الواجهة.

1. في نافذة طرفية جديدة (أو في النافذة الأصلية)، تأكد من أن البيئة الافتراضية مفعلة (`source venv/bin/activate`).
    
2. قم بتشغيل تطبيق الواجهة باستخدام Flask:
    
    ```
    flask --app ui/app run --port 5001
    ```
    
3. افتح المتصفح وادخل إلى العنوان `http://127.0.0.1:5001` للبدء في استخدام محرك البحث.
