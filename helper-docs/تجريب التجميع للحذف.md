### كيف تتحقق من أن التجميع (Clustering) يعمل؟

هذا سؤال عملي ومهم جداً. بما أن التجميع يتم في الخلفية، نحتاج إلى طريقة بسيطة للتحقق من نتائجه. إليك الخطوات:

1. **شغل خدمة بناء العناقيد:** بعد بناء النماذج الأساسية لمجموعة بيانات معينة (وليكن `antique`)، استخدم Postman (أو `curl`) لإرسال طلب `POST` إلى الـ Endpoint الجديد:
    
    - **URL:** `http://127.0.0.1:8002/build-clusters/`
        
    - **Body:**
        
        JSON
        
        ```
        {
            "dataset_name": "antique",
            "num_clusters": 20 
        }
        ```
        
2. **راقب السجل:** ستظهر رسالة في `representation.log` تخبرك بأن العملية قد بدأت، وبعد فترة (قد تستغرق بضع دقائق)، ستظهر رسالة أخرى تفيد بانتهاء العملية وحفظ ملف `clusters.joblib`.
    
3. **الاختبار والتحقق (باستخدام Notebook):**
    
    - افتح أي ملف Notebook (أو أنشئ ملفاً جديداً).
        
    - أضف خلية جديدة وانسخ فيها الكود التالي وقم بتشغيله:
        
    
    Python
    
    ```
    import joblib
    import os
    from config import MODELS_DIR
    
    # حدد مجموعة البيانات التي قمت بتجميعها
    dataset_name = 'antique'
    
    # تحميل النتائج
    model_dir = os.path.join(MODELS_DIR, dataset_name.replace('/', '_'))
    clusters_path = os.path.join(model_dir, 'clusters.joblib')
    doc_ids_path = os.path.join(model_dir, 'doc_ids.joblib')
    
    if os.path.exists(clusters_path) and os.path.exists(doc_ids_path):
        clusters = joblib.load(clusters_path)
        doc_ids = joblib.load(doc_ids_path)
    
        print(f"Clustering results loaded successfully for {dataset_name}.")
        print(f"Total documents: {len(doc_ids)}")
        print(f"Total clusters assigned: {len(clusters)}")
    
        # عرض العنقود الخاص بأول 10 مستندات كعينة
        print("\\n--- Sample Cluster Assignments ---")
        for i in range(10):
            print(f"Document ID '{doc_ids[i]}' is in Cluster: {clusters[i]}")
    else:
        print("Error: clusters.joblib or doc_ids.joblib not found. Please run the clustering process first.")
    ```
    
    إذا تم عرض أرقام العناقيد للمستندات، فهذا هو الدليل القاطع على أن العملية تمت بنجاح.
    

### رابعاً: ملف لاختبار التحسين الحاصل بعد التجميع

هذه هي النقطة الأهم. التجميع بحد ذاته لا يحسن مقياس MAP مباشرة، بل هو يفتح الباب أمام **تحسين تجربة المستخدم وعرض النتائج**. أفضل طريقة لإظهار هذا "التحسين" هي عبر تطبيق ميزة **"تصفح حسب الموضوع"** أو **"إيجاد المستندات المشابهة"**.

لقد قمت بإعداد Notebook جديد ومخصص لهذه المهمة. هو لا يقيس MAP، بل **يوضح كيف يمكننا استخدام نتائج التجميع بشكل عملي لتحسين النظام**.
