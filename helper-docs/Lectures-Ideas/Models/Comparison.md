This table summarizes the main idea, advantages (pros), and disadvantages (cons) of each model, from the foundational Boolean model to advanced Hybrid systems.

| Model                        | Main Idea                                                                                                                                                                                                        | Pros (Advantages)                                                                                                                                                                                                                                                                                                                                                                                                                        | Cons (Disadvantages)                                                                                                                                                                                                                                                                                                                                                                     |
| :--------------------------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Boolean Model (BoW)**      | Represents documents as a "Bag of Words" (word order doesn't matter). It performs strict matching using Boolean operators like `AND`, `OR`, `NOT` to find documents.                                             | • **Simple & Fast:** Easy to understand and implement for exact queries.&lt;br>• **Precise Control:** Gives users explicit control over which terms must appear in the results.                                                                                                                                                                                                                                                          | • **No Ranking:** Provides an "all-or-nothing" set of results with no order of relevance .&lt;br>• **No Term Weighting:** Treats all words as equally important .&lt;br>• **Rigid:** Can be difficult for users to formulate complex queries effectively.                                                                                                                                |
| **Vector Space Model (VSM)** | Represents documents and queries as numerical vectors in a high-dimensional space. It ranks documents by calculating the similarity (e.g., Cosine Similarity) between the query vector and each document vector. | • **Ranked Results:** Overcomes the biggest flaw of the Boolean model by providing a ranked list of relevant documents .&lt;br>• **Term Importance (TF-IDF):** Weights terms based on their importance in a document relative to the whole collection.&lt;br>• **Partial Matching:** Finds relevant documents even if they don't contain all query terms.                                                                                | • **Assumes Term Independence:** Doesn't understand the semantic meaning or relationship between words (e.g., "car" and "automobile" are seen as completely different).&lt;br>• **Length Bias:** Can favor longer documents since they have more words and higher term counts.&lt;br>• **Computationally Intensive:** Can be slow with very large vocabularies and document collections. |
| **BM25 (Best Matching 25)**  | An advanced probabilistic ranking function that improves upon TF-IDF. It's widely used in modern search engines.                                                                                                 | • **Handles Document Length:** Normalizes scores based on document length, removing the bias that VSM has towards longer documents .&lt;br>• **Smart Term Frequency:** Uses a "saturation factor" so that after a certain point, a term appearing more frequently doesn't disproportionately increase the score .&lt;br>• **High Relevance & Speed:** Generally provides better relevance than standard VSM and is very fast and robust. | • **Keyword-Based:** Still fundamentally relies on keyword matching and can miss the semantic context of a query.&lt;br>• **Parameter Tuning:** Requires tuning parameters (k_1 and b) to achieve the best performance for a specific dataset.                                                                                                                                           |
| **Hybrid Models**            | Combines multiple retrieval models (e.g., BM25 and a neural model) to get more accurate results. This can be done in parallel (fusing results) or in series (re-ranking).                                        | • **Best of Both Worlds:** Merges the speed and robustness of keyword models like BM25 with the deep contextual understanding of neural models .&lt;br>• **Highest Accuracy:** Captures both keyword relevance and semantic meaning, leading to significant improvements in precision and recall .&lt;br>• **Flexible:** Can be designed in many ways (e.g., fusion, re-ranking) to fit specific needs.                                  | • **Very Complex:** Significantly harder to design, implement, and maintain than single-model systems.&lt;br>• **Computationally Expensive:** Requires more resources, especially for the re-ranking stages that use large, slow neural models.&lt;br>• **Data Hungry:** The neural components often require large amounts of training data to perform well.                             |
