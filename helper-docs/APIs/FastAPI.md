- كل خدمة رئيسية تعمل الآن كتطبيق API مستقل، والتواصل بين الخدمات يتم حصرياً عبر طلبات HTTP.
- باستخدام إطار عمل FastAPI الذي يتميز بالسرعة وسهولة توثيق الـ Endpoints تلقائياً). ستعمل كل خدمة على منفذ (port) مختلف، وسيكون التواصل بينها حصرياً عبر طلبات HTTP (باستخدام مكتبة requests)، مما يضمن العزل التام ويجعل اختبار كل خدمة على حدة باستخدام Postman ممكناً.
